# Постановка задачи

В задачах **multi-series forecasting** временные ряды часто имеют различные масштабы значений, что может усложнять обучение глобальной модели. Для уменьшения различий между рядами применяется **нормализация значений временного ряда**.

Однако влияние нормализации может зависеть от класса модели.

В данном исследовании проверяется влияние различных способов нормализации на качество моделей прогнозирования.

---

# Гипотеза исследования

Нормализация по рядам важна для глобальных нейронных моделей, но влияет неоднозначно для моделей различных классов:

- для **нейронных сетей** нормализация может существенно улучшить обучение;
- для **моделей на деревьях решений** влияние будет слабым;
- для **простых бейзлайнов** нормализация не окажет заметного эффекта.

Для проверки гипотезы проводится сравнение моделей при различных способах масштабирования данных.

# Описание данных

В эксперименте использовался **случайный набор из 200 временных рядов** с гранулярностью **день** из датасета **M4**.

Каждый ряд имеет структуру:

- `unique_id` — идентификатор временного ряда
- `ds` — временная отметка
- `y` — значение ряда

### Разделение данных

Для оценки моделей использовалось разделение на **train / test**:

- последние **50 наблюдений** каждого ряда — **тестовый горизонт**
- остальные наблюдения — **обучающая выборка**

Модели должны предсказать **50 будущих значений**.

# Обучение моделей

Модели обучались на **обучающей выборке**, после чего использовались для прогнозирования **тестового горизонта**.

# Оценка качества

Для оценки качества использовались две метрики.

## sMAPE

Относительная ошибка прогнозирования.

Преимущества:

- инвариантность к масштабу
- устойчивость к различным диапазонам значений

---

## MASE

Нормализованная метрика, основанная на сравнении с наивным прогнозом.

Интерпретация:

- **MASE < 1** — модель лучше наивного прогноза
- **MASE > 1** — модель хуже

Использование двух метрик позволяет оценить как **относительную**, так и **масштабированную абсолютную ошибку**.

---

# Описание моделей

### Naive
Прогноз равен последнему наблюдаемому значению ряда.

### Seasonal Naive
Использует сезонность временного ряда.

### CatBoost
Модель **градиентного бустинга по деревьям решений**.

Для обучения использовались лаговые признаки временного ряда:

- лаги последних наблюдений (14 дней)
- лаги недельной сезонности

### PatchTST
Нейросетевая модель на основе **архитектуры Transformer**, предназначенная для прогнозирования временных рядов.

Особенности модели:

- patch-разбиение временного ряда
- глобальное обучение на множестве рядов

# Анализ результатов

**Бейзлайн-модели**

Практически не изменили качество при применении нормализации. Это объясняется тем, что они используют только значения временного ряда и не обучают параметры модели.

**CatBoost**

Различия между вариантами scaling оказались незначительными. Это соответствует теоретическим ожиданиям: деревья решений являются инвариантными к линейному масштабированию признаков.

**PatchTST**

Влияние масштабирования оказалось более заметным, однако в данном эксперименте наилучшие результаты были достигнуты **без дополнительной нормализации**.

Это может объясняться тем, что PatchTST уже содержит встроенные механизмы нормализации временных рядов.

---

# Выводы

Проведённый эксперимент позволяет сделать следующие выводы:

- влияние нормализации зависит от класса модели;
- бейзлайн-модели практически не реагируют на масштабирование данных;
- CatBoost демонстрирует устойчивость к нормализации благодаря использованию деревьев решений;
- PatchTST может быть чувствителен к нормализации, однако эффект зависит от архитектуры модели;
- дополнительная нормализация **не всегда приводит к улучшению качества прогнозирования**.

Гипотеза о различном влиянии нормализации на модели разных классов **частично подтверждается**.
